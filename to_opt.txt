Задача - оптимизировать процесс репликации данных из системы источника в подобие хранилища.
Реплицируется несколько сущностей, 3NF.

Текущая реализация на собственном языке, надо перенести на PL/SQL.
Будет использоваться на нескольких БД со слабым IO. Необходимо минимизировать число чтений.

Текущая реализация - записи таблицы источника считываются последовательно. Все ссылки записи на связанные таблицы перекодируются в коды системы-приемника.
Таблица перекодировки одна для всех сущностей, с комплексным ключом (тип объекта, ID объекта)

Цель - создание единый механизм кэша объектов для минимизации числа обращений к БД. 
Предполагается наполнять кэш ID объектов, затем пакетно заполнять его данными из таблицы перекодировки и, при необходимости, базовых таблиц связанных сущностей.
Предоставить интерфейс к полученным данным, механизм очистки кэша, механизм обработки ошибок чтения (отсутствие ID в таблице перекодировки)

В пакетном режиме сначала формируется список ID записей через последовательность вызовов процедуры ..
При каждом вызове процедура проверяет уникальность переданного ID, после чего добавляет его в список. Если ID уже есть в списке, 
никаких действий не требуется.
Затем вызывается процедура заполнения кэша, записи из списка считываются в коллекцию.
Дальше два варианта - либо оставляем исходную коллекцию и процедура чтения каждый раз перебирает ее до нужного элемента,
либо она преобразуется в ассоциативный массив с индексацией по ID. После этого в целом процедура чтения не нужна. 
Надо оценить расходы в обоих вариантах.   

Предусмотреть отключаемое логирование для оценки производительности, завязать на атрибут.

в объекте должны быть
0. очистка данных 
1. чтение одной записи
2. добавление записи по ID, забивает в последовательность через дистинкт
* 3. добавление набора записей через список int ID. 
* 4. добавление набора записей через список str ID?

5. атрибут отложенной загрузки. 
6. метод массовой загрузки кэша. Результат - коллекция ошибок. и флаг, всё ли загрузилось
7. атрибут лимита кэша
8. атрибут текущего количества записей.

11. переопределяемая запись - единица хранения
12. переопределяемые курсоры в заголовке для чтения данных
